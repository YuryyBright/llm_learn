````markdown
# Пайплайн класифікації текстів (RoBERTa + ONNX)

Цей проєкт містить повний цикл підготовки, навчання, оптимізації та тестування моделі для класифікації українських текстів на 4 категорії: **noise**, **list**, **preference** та **knowledge**.

В основі лежить україномовна модель `youscan/ukr-roberta-base`, яка донавчається (fine-tuning) на ваших даних, а потім оптимізується за допомогою технології ONNX та квантування до формату INT8 для максимального прискорення роботи на CPU без втрати якості.

---

## 🛠 Вимоги до оточення (Залежності)

pip install torch transformers datasets scikit-learn optimum[onnxruntime] matplotlib seaborn
```
````

## 🚀 Порядок запуску (Крок за кроком)

Нижче наведено правильну послідовність виконання скриптів із детальним описом того, що робить кожен із них.

### Крок 1. Генерація додаткових тестових даних

**Скрипт:** `generate_new_test_data.py`

- **Що робить:** Створює синтетичний набір реалістичних тестових даних (по 10+ прикладів для кожної категорії: робочі задачі, списки покупок, вподобання та енциклопедичні факти). Дані перемішуються та зберігаються.
- **Результат:** Створюється файл `new_test_data.jsonl`, який буде використаний на фінальному етапі для перевірки моделі.
- **Запуск:**
  ```bash
  python generate_new_test_data.py
  ```

### Крок 2. Навчання моделі (Fine-Tuning)

**Скрипт:** `train.py`

- **Що робить:** 1. Зчитує ваші базові датасети (`dataset_knowledge.jsonl`, `dataset_noise_list.jsonl`, `dataset_preference.jsonl`). 2. Розбиває їх на тренувальну (80%), валідаційну (10%) та тестову (10%) вибірки. 3. Токенізує тексти та запускає процес навчання за допомогою Hugging Face `Trainer`. 4. По завершенню оцінює модель на тестовій вибірці, виводить детальний звіт (Accuracy, F1-score) та будує графік `confusion_matrix.png`.
- **Результат:** Збережені ваги найкращої моделі у папці `./checkpoints` (наприклад, `./checkpoints/checkpoint-55`).
- **Запуск:**
  ```bash
  python train.py
  ```

### Крок 3. Експорт та квантування моделі (Оптимізація)

**Скрипт:** `export_and_quantize.py`

- **Що робить:** Бере навчену PyTorch-модель із папки `./checkpoints` і конвертує її у формат ONNX (FP32). Після цього застосовує динамічне квантування, перетворюючи ваги моделі у формат INT8. Це дозволяє зменшити розмір моделі майже в 4 рази.
- **Результат:** \* Базова ONNX-модель у папці `./chunk-classifier-onnx`.
  - Квантована INT8-модель у папці `./chunk-classifier-onnx-int8`.
- **Запуск:** _(перед запуском перевірте, чи вірний шлях до `CHECKPOINT_PATH` у коді)_
  ```bash
  python export_and_quantize.py
  ```

### Крок 4. Бенчмарк швидкості та стабільності

**Скрипт:** `test_inference.py`

- **Що робить:** Проганяє 100 тестових текстів через три версії моделі: оригінальну PyTorch, базову ONNX та квантовану ONNX INT8. Заміряє час обробки (мс на текст та кількість текстів на секунду). Також порівнює передбачення PyTorch та INT8-моделі, щоб переконатися, що квантування не зламало логіку класифікації.
- **Результат:** Вивід у консоль порівняльної статистики швидкості та повідомлення про відсутність/наявність розбіжностей у передбаченнях.
- **Запуск:**
  ```bash
  python test_inference.py
  ```

### Крок 5. Фінальна перевірка на нових даних

**Скрипт:** `test_new_data.py`

- **Що робить:** Завантажує оптимізовану ONNX-модель та файл `new_test_data.jsonl` (згенерований на Кроці 1). Робить передбачення на цих нових даних, розраховує підсумкову точність (Accuracy) і виводить приклади, де модель могла помилитися. Це фінальний "іспит" для моделі перед використанням у продакшені.
- **Результат:** Звіт про якість роботи моделі на раніше небачених даних.
- **Запуск:**
  ```bash
  python test_new_data.py
  ```
